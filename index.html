Ответы на вопросы

Что такое git workflow?

Это процесс,с помощью которого команда управляет изменения в коде.

Как он работает?

Он создает ветки,делает слияния и развертывает,чтобы она нормально работало.

Для чего он нужен?

Он нужен,чтобы улучшить работу в команде и сделать легче версии кодов.

Что такое gitrebase?

Он берет все изменения и как бы переписывает их поверх основного списка задач,чтобы они стояли в очереди после всех задач из основного списка.

Что такое git fetch?

Это как обновление,где ты загружаешь все новые изменения из удаленного репозитория,но ничего не меняешь у себя.

В чем отличие от gitrebase от gitmerge?

git merge сохраняет структуру веток, а git rebase делает историю более прямолинейной.

В чем отличие от gitfetch от gitpull?

fetch не вносит изменения ветку,а pull изменяет все

Что такое Github actions?

Он может создавать рабочие процессы,которые сразу же выполняют определённые действия при наступлении определённых событий,наапример обновление кода,создание pull request или же добавление тегов.
